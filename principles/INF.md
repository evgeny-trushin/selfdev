# INF: Infrastructure Readiness

**Category:** Operations Principles

Development quality includes runtime readiness:
- Check environment variables, secrets handling, CI/CD, migrations, health checks, deploy config, rollback path, and scaling assumptions
- Treat configuration drift and missing operational safeguards as first-class defects
- Prefer small infrastructure prompts tied to a single risk surface
- Production-critical flows require explicit readiness evidence, not assumptions
- Infrastructure findings should connect to the user or system impact they can cause
