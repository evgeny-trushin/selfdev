# UIX: UI State Coverage

**Category:** User Experience Principles

User-facing work is not complete until the interface behaves well across states:
- Check layout, hierarchy, copy, responsiveness, accessibility, keyboard behavior, and perceived performance
- Verify empty, loading, success, validation-error, and system-error states
- Evaluate whether the UI explains backend failures clearly instead of leaking raw errors
- Link UI prompts to the exact view, component, and user journey step they affect
- Prefer evidence from concrete screens and interactions over generic "improve UX" requests
