# Increment 0003: Evolutionary Tracking

**Requirement ID:** R3

## Description

The system must:
- Record fitness scores per generation
- Detect regressions between generations
- Expose history for trend analysis

## Related Principles

- [E1 — Small Mutations](../how/E1.md): Prefer many small changes over few large ones
- [E2 — Selection Pressure](../how/E2.md): Only accept changes that improve fitness
- [E3 — Memory](../how/E3.md): Track which prompts led to improvements
- [B4 — Feedback Loops](../how/B4.md): Negative feedback for stability, positive for growth
- [M3 — Transparency](../how/M3.md): All decisions are logged and explainable
